## 二、第一层：源资源组织（设计期）

### 2.1 资产管线的三个阶段

这是从 Naughty Dog（《最后生还者》《神秘海域》）到独立游戏都在用的模型：

```
源资源 (Source) → 中间资源 (Intermediate) → 最终资源 (Final)

Source:    PSD, .blend, .max, 原始音频    ← 只有这些进版本控制
           ↓  导出/转换
Intermediate:  FBX, PNG(全分辨率), WAV     ← 开放格式，可重新处理
           ↓  压缩/优化/平台转换
Final:    .glb(Draco), KTX2, WebP, Opus   ← 游戏引擎实际加载的
```

**核心原则：**
> "删除所有Final资源后，仅凭Source资源应该能完全重建整个游戏。"

### 2.2 目录组织 — AAA 工作室的黄金法则

**❌ 错误分组方式：按资源类型**
```
assets/
├── textures/    → 依赖关系混乱，跨类型引用难以追踪
├── models/
├── sounds/
└── materials/
```

**✅ 正确分组方式：按使用场景**
```
assets/
├── core/              ← 全局共享（引擎启动就需要的）
│   ├── shaders/
│   ├── fonts/
│   ├── crosshairs/
│   └── ui-common/
│
├── weapons/           ← 武器系统相关所有资源
│   ├── ak47/
│   │   ├── model.glb
│   │   ├── textures/
│   │   ├── sounds/
│   │   └── data.json         ← 武器的所有参数
│   ├── shotgun/
│   └── pistol/
│
├── environment/       ← 环境场景资源
│   ├── shooting_range/
│   │   ├── structure.glb
│   │   ├── textures/
│   │   ├── props/            ← 场景道具
│   │   └── lighting.json
│   └── arena/
│
├── enemies/           ← 敌人系统（自包含）
│   ├── zombie/
│   │   ├── model.glb
│   │   ├── animations/
│   │   ├── textures/
│   │   ├── sounds/
│   │   └── ai-config.json
│   └── soldier/
│
├── ui/                ← UI系统
│   ├── hud/
│   ├── menus/
│   └── effects/
│
└── audio/             ← 公共音效
    ├── music/
    └── ambient/
```

**为什么这么分？**
- 每个功能模块是**自包含**的，删除一个模块不影响其他
- 按需加载时可以直接加载整个文件夹
- 版本控制中冲突少（不同团队改不同文件夹）
- **依赖关系一目了然**

### 2.3 命名规范

```
资源命名 = [类型前缀]_[所属模块]_[描述]_[变体]

模型:    m_ak47_body.glb, m_zombie_head.glb
贴图:    t_ak47_diffuse.webp, t_floor_metal_n.png  (d=diffuse, n=normal, r=roughness)
音效:    s_ak47_shot_01.opus, s_ui_click.opus
动画:    a_zombie_walk.glb, a_zombie_attack.glb
数据:    d_weapons.json, d_enemies.json
UI:      ui_hud_crosshair.svg, ui_menu_bg.png

关键规则：
1. 全小写 + 下划线（跨平台兼容）
2. 禁止空格和特殊字符
3. 变体用数字后缀而非描述词（shot_01 > shot_loud）
4. 路径深度 ≤ 5 层
```

### 2.4 版本控制策略

| 资源类型 | 版本控制方案 | 原因 |
|----------|-------------|------|
| 源代码 (.ts/.js) | **Git** | 文本文件，可diff，可merge |
| 源资源 (.blend/.psd) | **Git LFS** | 二进制大文件，需要锁定 |
| 导出资源 (.glb/.webp) | **Git LFS** 或 不进仓库 | 可从源重建 |
| 配置文件 (.json) | **Git** | 文本文件，需要diff |
| 音效 (.opus/.wav) | **Git LFS** | 二进制 |

**AAA 工作室（使用 Perforce）vs 我们（使用 Git LFS）：**
```
Perforce (大型工作室):          Git LFS (小型/个人):
- 文件锁定防止冲突               - 大文件指针存储在Git
- 专门为二进制优化               - 免费，GitHub/GitLab支持
- 需要服务器                     - 适合资产 < 10GB 的项目
```

### 3.2 资产打包策略

**借鉴 Unity Addressables 和 YooAsset 的设计：**

```
打包原则：
1. 按 loadGroup 分bundle，不是按文件类型
2. 共享依赖单独提取（shader、公共材质）
3. 每个bundle ≤ 5MB（浏览器友好）
4. bundle内资源"同时加载、同时卸载"

示例打包方案：

environment_range.bundle (8MB) ← 训练场场景
ui.bundle            (1MB)     ← 所有UI资源
audio_common.bundle  (2MB)     ← 公共音效
audio_weapons.bundle (3MB)     ← 武器音效
```

## 四、第三层：运行时资源管理（核心）

### 4.1 分层加载架构

这是 AAA 游戏中最重要的运行时架构模式：

```
Layer 0: Bootstrap (启动层)           ← 始终在内存，永不卸载
  ├── 核心渲染管线
  ├── 基础shader
  ├── Loading UI
  └── 输入系统

Layer 1: Core Gameplay (核心玩法层)    ← 进入游戏后加载
  ├── 当前武器模型+音效
  ├── 玩家模型/动画
  ├── UI系统 (HUD)
  └── 基础环境

Layer 2: Active Level (当前关卡层)     ← 关卡切换时加载/卸载
  ├── 场景几何体
  ├── 敌人模型
  ├── 道具模型
  └── 关卡专属音效

Layer 3: Streaming (流式加载层)        ← 按需加载
  ├── 远处LOD资源
  ├── 稀有武器
  └── 特效变体
```

### 4.2 AssetManager 核心实现

综合 Unity Addressables、YooAsset、threejs-app 的最佳设计：

```typescript
// ========================================
// 核心数据结构
// ========================================

interface AssetHandle<T> {
  readonly id: string;
  readonly address: string;
  readonly asset: T;
  readonly status: 'loading' | 'loaded' | 'failed';
  readonly progress: number;        // 0-1
  refCount: number;
  release(): void;                  // 减引用计数
}

interface LoadRequest {
  address: string;
  type: 'texture' | 'model' | 'audio' | 'json';
  priority: 'critical' | 'high' | 'medium' | 'low';
  onComplete?: (handle: AssetHandle<any>) => void;
  onProgress?: (progress: number) => void;
  onError?: (error: Error) => void;
  timeout?: number;
}

// ========================================
// AssetManager — 全局单例
// ========================================

class AssetManager {
  // 核心数据结构
  private cache: Map<string, AssetHandle<any>> = new Map();
  private loadingQueue: PriorityQueue<LoadRequest> = new PriorityQueue();
  private loaderMap: Map<string, IAssetLoader> = new Map();

  // 加载中跟踪
  private activeLoads: Map<string, Promise<AssetHandle<any>>> = new Map();
  private maxConcurrentLoads: number = 4;

  // 统计
  private stats = {
    totalLoaded: 0,
    totalBytesLoaded: 0,
    cacheHits: 0,
    cacheMisses: 0,
    failedLoads: 0,
  };

  // ========================================
  // 公共API
  // ========================================

  /**
   * 异步加载资源（自动去重、自动引用计数）
   * 如果资源已在缓存中，立即返回（零开销）
   * 如果正在加载中，返回同一个Promise（避免重复加载）
   */
  async load<T>(address: string, options?: Partial<LoadRequest>): Promise<AssetHandle<T>> {
    // 1. 缓存命中 → 直接返回，增加引用计数
    const cached = this.cache.get(address);
    if (cached && cached.status === 'loaded') {
      cached.refCount++;
      this.stats.cacheHits++;
      return cached as AssetHandle<T>;
    }

    // 2. 正在加载中 → 返回同一个Promise（去重）
    if (this.activeLoads.has(address)) {
      return this.activeLoads.get(address)! as Promise<AssetHandle<T>>;
    }

    // 3. 开始加载
    this.stats.cacheMisses++;
    const promise = this.performLoad<T>(address, options);
    this.activeLoads.set(address, promise);

    try {
      const handle = await promise;
      return handle;
    } finally {
      this.activeLoads.delete(address);
    }
  }

  /**
   * 预加载资源（静默加载，不阻塞）
   */
  async preload(addresses: string[], priority: 'high' | 'medium' | 'low' = 'medium'): Promise<void> {
    const promises = addresses.map(addr =>
      this.load(addr, { priority }).catch(err => {
        console.warn(`[AssetManager] Preload failed for ${addr}:`, err);
        return null;
      })
    );
    await Promise.all(promises);
  }

  /**
   * 加载整个资源组（按loadGroup）
   */
  async loadGroup(groupName: string, onProgress?: (p: number) => void): Promise<void> {
    const group = this.manifest.loadGroups[groupName];
    if (!group) throw new Error(`Unknown load group: ${groupName}`);

    let loaded = 0;
    const total = group.assets.length;

    await Promise.all(
      group.assets.map(addr =>
        this.load(addr, {
          onProgress: () => {
            loaded++;
            onProgress?.(loaded / total);
          }
        })
      )
    );
  }

  /**
   * 释放引用（引用计数归零后自动卸载）
   */
  release(address: string): void {
    const handle = this.cache.get(address);
    if (!handle) return;

    handle.refCount = Math.max(0, handle.refCount - 1);

    if (handle.refCount === 0) {
      this.unload(address);
    }
  }

  // ========================================
  // 内部实现
  // ========================================

  private async performLoad<T>(address: string, options?: Partial<LoadRequest>): Promise<AssetHandle<T>> {
    const assetInfo = this.manifest.assets[address];
    if (!assetInfo) throw new Error(`Asset not in manifest: ${address}`);

    // 1. 先加载依赖
    if (assetInfo.dependencies?.length) {
      await Promise.all(assetInfo.dependencies.map(dep => this.load(dep)));
    }

    // 2. 选择加载器
    const loader = this.getLoader(assetInfo.type);
    if (!loader) throw new Error(`No loader for type: ${assetInfo.type}`);

    // 3. 带重试的加载
    const asset = await this.loadWithRetry(loader, address, assetInfo, options);

    // 4. 创建Handle，存入缓存
    const handle: AssetHandle<T> = {
      id: generateId(),
      address,
      asset,
      status: 'loaded',
      progress: 1.0,
      refCount: 1,
      release: () => this.release(address),
    };

    this.cache.set(address, handle);
    this.stats.totalLoaded++;
    this.stats.totalBytesLoaded += assetInfo.size.total;

    return handle;
  }

  /**
   * 带指数退避的重试加载
   */
  private async loadWithRetry<T>(
    loader: IAssetLoader,
    address: string,
    info: AssetInfo,
    options?: Partial<LoadRequest>,
    maxRetries = 3
  ): Promise<T> {
    for (let attempt = 0; attempt < maxRetries; attempt++) {
      try {
        const timeout = options?.timeout ?? 30000;
        return await this.loadWithTimeout(loader.load(info), timeout);
      } catch (error) {
        if (attempt === maxRetries - 1) {
          this.stats.failedLoads++;
          options?.onError?.(error as Error);
          throw error;
        }
        // 指数退避: 1s, 2s, 4s
        await new Promise(r => setTimeout(r, 1000 * Math.pow(2, attempt)));
      }
    }
    throw new Error(`Failed to load ${address} after ${maxRetries} attempts`);
  }

  /**
   * 超时控制
   */
  private loadWithTimeout<T>(promise: Promise<T>, timeout: number): Promise<T> {
    return Promise.race([
      promise,
      new Promise<T>((_, reject) =>
        setTimeout(() => reject(new Error('Load timeout')), timeout)
      ),
    ]);
  }

  /**
   * 卸载资源（释放内存）
   */
  private unload(address: string): void {
    const handle = this.cache.get(address);
    if (!handle) return;

    // 释放GPU资源
    const asset = handle.asset as any;
    if (asset?.dispose) {
      asset.dispose();           // Three.js 对象
    } else if (asset?.destroy) {
      asset.destroy();           // 通用销毁
    }

    this.cache.delete(address);
  }

  /**
   * 获取内存使用统计
   */
  getMemoryUsage(): { cacheSize: number; totalBytes: number; activeRefs: number } {
    let totalBytes = 0;
    let activeRefs = 0;

    for (const [addr, handle] of this.cache) {
      const info = this.manifest.assets[addr];
      if (info) totalBytes += info.size.total;
      if (handle.refCount > 0) activeRefs++;
    }

    return {
      cacheSize: this.cache.size,
      totalBytes,
      activeRefs,
    };
  }
}
```

### 4.3 引用计数详解

这是资源不泄露的**核心机制**：

```
场景：两个实体同时引用一把AK47模型

时间线：
t0: Player.LoadWeapon('ak47')    → ak47.RefCount = 1
t1: Enemy.LoadWeapon('ak47')     → ak47.RefCount = 2   (同一handle，不重复加载!)
t2: Player.SwitchWeapon('pistol') → ak47.RefCount = 1   (Player释放)
t3: Enemy.Die()                  → ak47.RefCount = 0   → 自动卸载!

关键规则：
1. Load时 +1
2. Release时 -1
3. RefCount = 0 时自动销毁
4. 同地址的多次Load返回同一个Handle（去重）
5. 依赖的资产也会被计数（加载AK47时其材质贴图也会+1）
```

### 4.4 优先级加载队列

```typescript
class PriorityLoadQueue {
  private queues = {
    critical: [] as LoadRequest[],   // 阻塞游戏进行
    high: [] as LoadRequest[],       // 当前帧必须
    medium: [] as LoadRequest[],     // 即将需要
    low: [] as LoadRequest[],        // 预加载/预测
  };

  // 优先级调度：critical > high > medium > low
  // 每帧最多处理N个请求，避免卡顿
  update(maxPerFrame: number = 2): void {
    let processed = 0;

    for (const priority of ['critical', 'high', 'medium', 'low']) {
      const queue = this.queues[priority];
      while (queue.length > 0 && processed < maxPerFrame) {
        const request = queue.shift()!;
        this.processRequest(request);
        processed++;
      }
    }
  }

  // 空闲时预加载低优先级资源
  processIdle(): void {
    if (this.queues.low.length > 0) {
      this.processRequest(this.queues.low.shift()!);
    }
  }
}
```

### 4.5 加载时机的四种请求类型

| 类型 | 优先级 | 触发时机 | 示例 |
|------|--------|----------|------|
| **Blocking** | 最高 | 游戏无法继续 | 启动Loading画面资源 |
| **Active** | 高 | 当前场景必须 | 当前武器模型、HUD |
| **Queued** | 低 | 有需要但可等待 | 切换武器的动画 |
| **Passive** | 最低 | 空闲预加载 | 预测玩家可能用的枪 |

## 五、Web 游戏特殊策略

### 5.1 浏览器三层存储架构

```
┌──────────────────────────────────────────────────┐
│  第一层：Memory Cache (最快，最不稳定)              │
│  RAM中的资源，页面刷新即丢失                        │
│  用途：当前帧正在使用的资源                          │
├──────────────────────────────────────────────────┤
│  第二层：Cache Storage API (快，持久)              │
│  Service Worker管理的HTTP缓存                      │
│  用途：已编译的代码、压缩后的资源bundle               │
│  策略：Cache-First (代码) / SWR (资源)              │
├──────────────────────────────────────────────────┤
│  第三层：IndexedDB (较慢，大容量，可查询)           │
│  结构化的本地数据库                                 │
│  用途：游戏存档、设置、离线操作队列                   │
│  策略：事务性写、索引查询                            │
└──────────────────────────────────────────────────┘
```

### 5.2 Service Worker 游戏资源缓存

```typescript
// sw.ts — 游戏Service Worker
const CACHE_NAMES = {
  core: 'fps-core-v2',        // 引擎代码、核心Shader
  assets: 'fps-assets-v2',    // 预加载的游戏资源
  dynamic: 'fps-dynamic-v1',  // 运行时动态缓存
};

// 安装时预缓存核心资源
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAMES.core).then(cache =>
      cache.addAll([
        '/',
        '/index.html',
        '/assets/bootstrap.bundle.js',
        '/assets/core-shaders.bundle.js',
      ])
    )
  );
});

// 请求拦截：智能缓存策略
self.addEventListener('fetch', (event) => {
  const url = new URL(event.request.url);

  // .bundle.js → Cache-First (静止不变的构建产物)
  if (url.pathname.endsWith('.bundle.js')) {
    event.respondWith(cacheFirst(event.request, CACHE_NAMES.core));
    return;
  }

  // .glb, .webp, .opus → Stale-While-Revalidate
  if (/\.(glb|webp|opus|ktx2)$/.test(url.pathname)) {
    event.respondWith(staleWhileRevalidate(event.request, CACHE_NAMES.assets));
    return;
  }

  // API调用 → Network-First
  event.respondWith(networkFirst(event.request));
});

// Cache-First: 缓存命中就不发网络请求
async function cacheFirst(request: Request, cacheName: string): Promise<Response> {
  const cached = await caches.match(request);
  if (cached) return cached;
  const response = await fetch(request);
  const cache = await caches.open(cacheName);
  cache.put(request, response.clone());
  return response;
}

// Stale-While-Revalidate: 立即返回缓存，后台更新
async function staleWhileRevalidate(request: Request, cacheName: string): Promise<Response> {
  const cached = await caches.match(request);
  const fetchPromise = fetch(request).then(response => {
    caches.open(cacheName).then(cache => cache.put(request, response.clone()));
    return response;
  });
  return cached ?? fetchPromise;
}
```

```typescript
class SaveManager {
  private db: IDBDatabase | null = null;

  async init(): Promise<void> {
    return new Promise((resolve, reject) => {
      const request = indexedDB.open('FPSGameDB', 1);

      request.onupgradeneeded = (event) => {
        const db = (event.target as IDBOpenDBRequest).result;

        // 存档表
        const saves = db.createObjectStore('saves', { keyPath: 'slotId' });
        saves.createIndex('lastModified', 'lastModified', { unique: false });

        // 设置表
        db.createObjectStore('settings', { keyPath: 'key' });

        // 统计数据表
        const stats = db.createObjectStore('stats', { keyPath: 'sessionId' });
        stats.createIndex('date', 'date', { unique: false });

        // 离线操作队列（Outbox Pattern）
        db.createObjectStore('pendingActions', {
          keyPath: 'id',
          autoIncrement: true,
        });
      };

      request.onsuccess = () => {
        this.db = request.result;
        resolve();
      };
      request.onerror = () => reject(request.error);
    });
  }

  // 事务安全地保存
  async saveGame(slotId: number, data: GameSaveData): Promise<void> {
    const tx = this.db!.transaction('saves', 'readwrite');
    tx.objectStore('saves').put({
      slotId,
      data,
      lastModified: Date.now(),
    });
    return new Promise((resolve, reject) => {
      tx.oncomplete = () => resolve();
      tx.onerror = () => reject(tx.error);
    });
  }

  // 离线队列（购买/上传分数等）
  async queueAction(action: OfflineAction): Promise<void> {
    const tx = this.db!.transaction('pendingActions', 'readwrite');
    tx.objectStore('pendingActions').add({
      ...action,
      timestamp: Date.now(),
      retryCount: 0,
    });
  }
}
```

## 六、UI 资源管理专项

### 6.1 UI 资源与游戏资源分离

```
UI资源管理原则：
1. UI资源始终在内存（或快速缓存中），不允许加载延迟
2. UI资源与游戏资源分开打包（ui.bundle）
3. UI预制体使用对象池（频繁显示/隐藏的元素）
4. Canvas分离：静态Canvas vs 动态Canvas

为什么Canvas要分离？
┌──────────────────────────────────┐
│  静态Canvas (StaticHUD)          │  ← 准星、血条、弹药 — 变化少
│  只在数据变化时更新               │     不触发全Canvas重绘
├──────────────────────────────────┤
│  动态Canvas (DynamicHUD)         │  ← 伤害数字、命中标记、击杀信息
│  每帧可能更新                     │     独立Canvas，不影响静态部分
└──────────────────────────────────┘
```

### 6.2 UI 对象池模式

```typescript
class UIPool<T extends UIElement> {
  private available: T[] = [];
  private active: T[] = [];
  private prefab: () => T;  // 工厂函数

  constructor(prefab: () => T, prewarmCount: number = 5) {
    this.prefab = prefab;
    // 预热：预先创建N个实例备用
    for (let i = 0; i < prewarmCount; i++) {
      const instance = this.prefab();
      instance.setVisible(false);
      this.available.push(instance);
    }
  }

  get(): T {
    let instance: T;
    if (this.available.length > 0) {
      // 从池中取 → O(1)
      instance = this.available.pop()!;
    } else {
      // 池空了 → 动态扩容
      instance = this.prefab();
    }
    instance.setVisible(true);
    this.active.push(instance);
    return instance;
  }

  release(instance: T): void {
    instance.setVisible(false);
    instance.reset();  // 重置状态
    const idx = this.active.indexOf(instance);
    if (idx !== -1) this.active.splice(idx, 1);
    this.available.push(instance);
  }

  releaseAll(): void {
    while (this.active.length > 0) {
      this.release(this.active[0]);
    }
  }
}

// 使用示例
const damageNumPool = new UIPool(() => new DamageNumber(), 10);
const hitmarkerPool = new UIPool(() => new Hitmarker(), 5);
```

### 7.2 加载进度追踪

```typescript
class LoadingProgress {
  private stages: LoadingStage[] = [];
  private currentStage = 0;

  addStage(name: string, bytes: number, weight: number): void {
    this.stages.push({ name, bytes, weight, loaded: 0, total: bytes });
  }

  updateStage(name: string, bytesLoaded: number): void {
    const stage = this.stages.find(s => s.name === name);
    if (stage) stage.loaded = bytesLoaded;
  }

  get overallProgress(): number {
    // 加权平均：大文件占更大的进度比重
    let totalWeight = 0;
    let weightedProgress = 0;

    for (const stage of this.stages) {
      const stageProgress = stage.total > 0 ? stage.loaded / stage.total : 0;
      weightedProgress += stageProgress * stage.weight;
      totalWeight += stage.weight;
    }

    return totalWeight > 0 ? weightedProgress / totalWeight : 0;
  }

  get currentStageName(): string {
    return this.stages[this.currentStage]?.name ?? '';
  }
}

// 使用
const progress = new LoadingProgress();
progress.addStage('引擎核心', 500000, 10);    // 权重10
progress.addStage('UI系统', 1000000, 15);     // 权重15
progress.addStage('武器', 3000000, 30);       // 权重30
progress.addStage('环境', 8000000, 35);       // 权重35
progress.addStage('音效', 2000000, 10);       // 权重10

manager.onProgress = (url, loaded, total) => {
  progress.updateStage(getStageName(url), estimateBytes(url));
  updateLoadingScreen(progress.overallProgress, progress.currentStageName);
};
```

## 八、错误处理与降级策略

### 8.1 分级降级体系

```typescript
class AssetFallbackSystem {
  /**
   * 当资源加载失败时，按优先级尝试替代方案
   */
  async loadWithFallback(address: string): Promise<any> {
    const fallbackChain = this.getFallbackChain(address);

    for (const fallback of fallbackChain) {
      try {
        const result = await this.assetManager.load(fallback);
        // 记录降级情况
        if (fallback !== address) {
          console.warn(`[Asset] Fallback: ${address} → ${fallback}`);
          this.logFallback(address, fallback);
        }
        return result;
      } catch (error) {
        continue;  // 尝试下一个备选
      }
    }

    // 全部失败 → 显示默认占位符
    return this.getDefaultPlaceholder(address);
  }

  private getFallbackChain(address: string): string[] {
    const chain = [address];  // 首先尝试原始地址

    // 根据资产类型添加备选
    if (address.includes('textures')) {
      chain.push(
        address.replace('.ktx2', '.webp'),   // 降级到WebP
        address.replace('4k', '2k'),          // 降分辨率
        'textures/default_diffuse.webp'        // 默认紫色格子贴图
      );
    }

    if (address.includes('models')) {
      chain.push(
        address.replace('.glb', '_lod2.glb'), // 低面数版本
        'models/placeholder_box.glb'           // 默认方块
      );
    }

    if (address.includes('sounds')) {
      chain.push(
        address.replace('.opus', '.mp3'),      // 格式降级
        'sounds/silence_100ms.opus'            // 静音
      );
    }

    return chain;
  }
}
```

### 8.2 超时与重试策略

```
网络请求的重试策略：
  - 第1次失败：立即重试
  - 第2次失败：等待 1秒 重试
  - 第3次失败：等待 2秒 重试
  - 第4次失败：等待 4秒 重试
  - 全部失败：降级到fallback

并发控制：
  - 最多同时4个网络请求（浏览器限制是6个/域名）
  - CDN资源用不同子域名（cdn1., cdn2.）绕过限制
  - 关键资源（bootstrap）超时时间 = 10秒
  - 普通资源超时时间 = 30秒
```

## 九、内存管理与泄漏检测

### 9.1 Web 游戏内存预算

```
浏览器内存预算（以FPS游戏为例）：

总计可用: ~500MB-1GB (桌面) / ~200MB (移动)

分配方案:
├── 引擎运行时:          ~30MB   (Three.js + 场景图)
├── 当前场景模型:        ~80MB   (环境+道具)
├── 武器模型+动画:       ~40MB   (所有可用武器)
├── 纹理(GPU):          ~100MB  (贴图占用GPU显存)
├── 音效(解码后):        ~30MB   (AudioBuffer)
├── UI+Canvas:          ~20MB
├── JS堆:               ~40MB   (游戏逻辑对象)
├── 预留/缓冲:           ~160MB
└── 未使用:             ~400MB+ (桌面余量)

危险信号:
  - 纹理总大小 > 200MB → 考虑KTX2压缩
  - 模型面数累计 > 500K → 实施LOD
  - AudioBuffer > 64MB → 使用流式音频
  - JS堆频繁GC → 检查对象池使用
```

### 9.2 内存泄漏检测

```typescript
class MemoryMonitor {
  private snapshots: MemorySnapshot[] = [];

  takeSnapshot(label: string): void {
    const mem = (performance as any).memory;
    this.snapshots.push({
      label,
      timestamp: performance.now(),
      jsHeapSize: mem?.usedJSHeapSize ?? 0,
      totalJSHeap: mem?.totalJSHeapSize ?? 0,
      cachedAssets: this.assetManager.getMemoryUsage(),
    });
  }

  detectLeaks(): LeakReport {
    // 检查：每次关卡切换后，JS堆是否持续增长
    const levelTransitions = this.getLevelTransitions();
    let leakSuspected = false;

    for (let i = 1; i < levelTransitions.length; i++) {
      const prev = levelTransitions[i - 1].jsHeapSize;
      const curr = levelTransitions[i].jsHeapSize;
      // 如果切换关卡后内存比之前高20%以上 → 可能泄漏
      if (curr > prev * 1.2) {
        leakSuspected = true;
        break;
      }
    }

    return {
      leakSuspected,
      activeHandles: this.assetManager.getMemoryUsage().activeRefs,
      orphanedHandles: this.findOrphanedHandles(),
      recommendation: leakSuspected
        ? '检查关卡切换时是否正确释放所有资源Handle'
        : '内存表现正常',
    };
  }
}
```

### 10.3 关键设计决策汇总

| 决策 | 方案 | 理由 |
|------|------|------|
| 资源分组 | **按使用场景**，不按类型 | 减少跨模块依赖，方便按需加载和卸载 |
| 引用管理 | **引用计数** | 多实体共享资源时防止过早或遗漏卸载 |
| 加载去重 | **Promise缓存** | 同一地址并发请求返回同一个Promise |
| 优先级调度 | **4级优先级队列** | 保证关键资源优先，空闲时预加载 |
| Web缓存 | **Cache API + IndexedDB** | 文件用Cache API，结构化数据用IndexedDB |
| 错误处理 | **多级Fallback链** | 降分辨率→降格式→默认占位符 |
| 启动速度 | **3阶段加载** | 先启动→再核心→后台加载其余 |
| UI资源 | **独立bundle + 对象池** | UI不能有延迟感，对象池避免GC |
| Bundle大小 | **≤ 5MB/个** | 浏览器友好，便于增量更新 |
| 内存监控 | **快照对比** | 关卡切换前后对比检测泄漏 |
| 版本控制 | **Git LFS (大文件)** | 免费方案，适合资产 < 10GB |
| 资源构建 | **自动化脚本** | manifest + 压缩 + bundle打包全自动 |
