# 🦁 ZooSim 3D — 动物园模拟器 项目总结

## 项目概览

**ZooSim 3D** 是一款基于浏览器的 3D 动物园经营模拟游戏。玩家可以建造围栏、饲养动物、铺设步道、布置装饰，吸引游客参观并赚取收入，逐步打造五星级动物园。

- **类型**：3D 模拟经营游戏
- **技术栈**：纯前端 HTML/CSS/JS + Three.js (r160)
- **文件结构**：单文件应用 (`zoo-simulator.html`，约 1413 行)
- **当前版本**：v1.5
- **开发周期**：2026-07-17 ~ 2026-07-19

## 功能模块

### 1. 建造系统

| 类别 | 内容 | 数量 |
|------|------|------|
| 🦁 动物 | 狮子、大象、老虎、熊猫、企鹅、长颈鹿、斑马、猴子、熊、鳄鱼、火烈鸟、考拉 | 12 种 |
| 🏗️ 设施 | 围栏、步道、小吃摊、洗手间、长椅、信息牌、入口门 | 7 种 |
| 🌳 装饰 | 橡树、松树、花丛、喷泉、狮子雕像、路灯、灌木、大岩石 | 8 种 |
| 🗺️ 地形 | 草地、草原、丛林、雪地、竹林、沼泽、森林地、沙地、水域 | 9 种 |
| 🔧 操作 | 拆除、保存、读取 | 3 种 |

### 2. 游客 AI 系统

游客采用**有限状态机 (FSM)** 驱动：

```
                    ┌──────────┐
           ┌──────▶│ entering  │ (从大门/边界出生)
           │       └────┬─────┘
           │            ▼
           │       ┌──────────┐
           │ ◀─────│ seeking   │ (选择目标)
           │       └────┬─────┘
           │            ▼
           │       ┌──────────┐
           │       │ walking   │ (走向目标)
           │       └────┬─────┘
           │            ▼
           │  ┌─────────────────────┐
           │  │ watching   │ queuing │ eating
           │  │ (看动物)   │ (排队)  │ (用餐)
           │  └─────┬─────┴────┬────┘
           │        └──────────┘
           │            ▼
           │       ┌──────────┐
           └───────│ leaving   │ (满意度低/时间到/闭园)
                   └──────────┘
```

- **满意度系统**：看动物 +0.06~0.08，票价影响基准满意度
- **步道偏好**：移动时优先走向步道
- **碰撞检测**：不穿越围栏、不进入水域
- **配额生成**：根据动物吸引力 + 设施加成计算每日游客配额
- **小吃摊消费**：30% 概率光顾，排队→购买→继续参观

### 3. 经济系统

| 项目 | 说明 |
|------|------|
| 初始资金 | $8,000 |
| 门票收入 | `游客数 × 票价` (票价 = 20 + 动物数×5) |
| 小吃收入 | `游客数 × 50% × $10` |
| 动物维护 | `Σ(动物upkeep × 0.4)` / 天 |
| 政策补贴 | +$1,000 / 天 |
| 动物售出 | 返还购买价的 50% |

### 4. 日夜循环

6 个时段平滑过渡（实时 ~90 秒/天 @ 1x）：

```
06:00 黎明 ──▶ 08:00 开园 ──▶ 12:00 正午 ──▶ 18:00 黄昏 ──▶ 20:00 闭园结算 ──▶ 22:00 夜晚
   (蓝紫)       (天蓝)        (亮白)        (橙黄)        (深蓝)          (暗蓝)
```

- 天空色 HSL 插值 + 雾色同步
- 太阳位置和强度正弦曲线
- 环境光同步调整

### 5. 动物园评级

| 星级 | 吸引力要求 |
|------|-----------|
| ⭐ | 0+ |
| ⭐⭐ | >80 |
| ⭐⭐⭐ | >250 |
| 🌟🌟🌟🌟 | >500 |
| 🌟🌟🌟🌟🌟 | >900 |

### 6. 围栏检测算法

使用 **BFS（广度优先搜索）** 自动识别围栏封闭区域：

1. 遍历所有非围栏、非水域格子
2. BFS 扩展连通区域
3. 触及边界的区域视为非封闭（不是围栏）
4. 封闭区域自动分配 biome（基于区域内动物类型）
5. 动物只能放置在围栏内，且 biome 需匹配

### 7. 存档系统

- 3 个存档槽位，基于 `localStorage`
- 自动保存（每 60 秒 + 关键操作触发）
- 存档内容：网格状态、动物列表、围栏信息、经济数据、时间
- 支持存档删除和"继续上次游戏"

## 技术亮点

1. **13 CDN 并行竞速加载**：从多个国内/国际 CDN 同时请求 Three.js，任一成功即初始化，兼顾国内外网络环境
2. **ES Module importmap 兜底**：当 script src 被封锁时，通过 importmap 方式加载
3. **预计算缓存**：大门位置、小吃摊位置、围栏周边格子预计算，避免每帧全图扫描
4. **累加器式游客生成**：用累加器 + 每帧上限控制游客生成速率，杜绝卡顿
5. **BFS 围栏检测**：自动识别封闭区域，支持任意形状围栏
6. **状态机游客 AI**：7 种状态 + 平滑移动 + 步道偏好 + 碰撞检测
7. **单文件架构**：零构建工具、零依赖安装，浏览器直接打开即可运行
